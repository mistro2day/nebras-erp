# دليل تشغيل وتهيئة منصة Nebras ERP

يركز هذا المستند على تقديم دليل شامل لتثبيت وإعداد وتشغيل منصة **Nebras ERP** (الباك إند والفرونت إند).

---

## 1. متطلبات النظام الأساسية (Prerequisites)
قبل البدء، تأكد من تثبيت البرمجيات التالية على جهازك:
* **Python 3.13** أو أحدث.
* **Node.js v20** أو أحدث مع **npm**.
* **PostgreSQL 17** (اختياري، المنصة تدعم SQLite افتراضياً للتطوير السريع).
* **Redis** (لتشغيل مهام Celery الخلفية والكاش).

---

## 2. إعداد وتشغيل الباك إند (Django Backend)

### أ. تثبيت الاعتماديات وإنشاء البيئة الافتراضية
1. افتح مبدل الأوامر في مجلد `backend` أو مجلد المشروع الرئيسي.
2. أنشئ بيئة عمل افتراضية وفعّلها:
   ```bash
   python -m venv .venv
   # لتفعيل البيئة في Windows (Powershell):
   .venv\Scripts\Activate.ps1
   # لتفعيل البيئة في Linux/macOS:
   source .venv/bin/activate
   ```
3. ثبّت المكتبات والاعتماديات المطلوبة:
   ```bash
   pip install -r backend/requirements.txt
   ```

### ب. تهيئة قاعدة البيانات والتهجيرات (Migrations)
قم بتشغيل الأوامر التالية لإنشاء الجداول والبيانات الأساسية:
```bash
python backend/manage.py migrate
```

### ج. تشغيل سيرفر الباك إند
لتشغيل خادم التطوير المحلي للباك إند (يعمل افتراضياً على المنفذ `8000`):
```bash
python backend/manage.py runserver
```

### د. تشغيل الاختبارات الفنية (Unit Tests)
للتحقق من سلامة الأكواد والتأكد من نجاح عمل الموديولات (مثل موديول النقل):
```bash
python backend/manage.py test apps.transport
```

---

## 3. إعداد وتشغيل الفرونت إند (Angular Frontend)

### أ. تثبيت الاعتماديات
1. انتقل إلى مجلد `frontend`:
   ```bash
   cd frontend
   ```
2. قم بتثبيت الحزم البرمجية والاعتماديات المطلوبة:
   ```bash
   npm install
   ```

### ب. تشغيل سيرفر الفرونت إند
لتشغيل خادم التطوير المحلي للفرونت إند (يعمل افتراضياً على الرابط `http://localhost:4200`):
```bash
npm start
```
أو عبر استخدام Angular CLI مباشرة:
```bash
npx ng serve
```

### ج. بناء نسخة الإنتاج (Production Build)
لبناء ملفات الإنتاج وتجهيزها للنشر:
```bash
npm run build:prod
```

---

## 4. البنية المعمارية وتكامل موديول النقل
* يعتمد موديول النقل على معايير Clean Architecture وDDD.
* يتكامل موديول النقل مع موديول المالية من خلال توليد قيود محاسبية تلقائية لتموين الوقود.
* يتكامل الموديول مع موديول الصيانة (CMMS) من خلال فتح بلاغات صيانة فورية عند فشل الفحوصات الفنية اليومية للحافلات.
