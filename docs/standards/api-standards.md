# معايير واجهات الـ API الموحدة (api-standards.md) - Nebras ERP

توفر هذه الوثيقة معايير وتوقيع الرد القياسي الموحد لجميع واجهات الـ REST API في منصة **Nebras ERP** لضمان سهولة الدمج مع الواجهات الأمامية وتطبيقات الجوال.

---

## 1. التنسيق القياسي للرد (Standard Response Format)

يجب أن تلتزم جميع واجهات المنصة بصيغة الرد الهيكلية التالية:

### رد النجاح (Success Response - HTTP 200/201)
```json
{
  "success": true,
  "message": "تمت العملية بنجاح.",
  "data": {
    "id": "c3b0ac48-d3c2-48f6-a70c-26ee3b1b9e59",
    "name": "اسم الكيان"
  }
}
```

### رد الفشل والخطأ (Error Response - HTTP 400/403/404/500)
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "هناك أخطاء في البيانات المرسلة.",
    "details": {
      "field_name": ["هذا الحقل مطلوب."]
    }
  }
}
```

---

## 2. معايير الصفحات والفلترة والترتيب (Pagination, Filtering, Sorting)

- **ترقيم الصفحات:** يجب استخدام صيغة ترقيم الصفحات الموحدة `StandardPagination` التي تعود بالبارامترات التالية:
  - `count`: إجمالي العناصر.
  - `next`: رابط الصفحة التالية.
  - `previous`: رابط الصفحة السابقة.
  - `results`: مصفوفة البيانات.
- **الفلترة والترتيب:** يجب عزل بارامترات الفلترة لتكون متطابقة مع الكيان، وتمرير معامل الترتيب عبر بارامتر `ordering` (مثال: `?ordering=-created_at`).
