# معايير تطوير الباك إند (backend-standards.md) - Nebras ERP

تحدد هذه الوثيقة المعايير والقواعد الصارمة التي يجب على كافة مطوري الباك إند الالتزام بها لضمان تكامل الأكواد واستقرار المعمارية.

---

## 1. هيكل الموديول البرمجي (Clean Architecture / DDD)

يجب تنظيم أي تطبيق Django جديد برمجياً باتباع الهيكل رباعي الطبقات:

- **domain (النطاق):** يحتوي على كيانات الأعمال النظيفة (`models.py`, `value_objects.py`, `exceptions.py`) ولا يعتمد على أي مكتبات خارجية.
- **application (التطبيق):** يحتوي على منطق العمل وحالات الاستخدام (`services.py`, `events.py`).
- **infrastructure (البنية التحتية):** يحتوي على كود الوصول لقاعدة البيانات والاتصال الخارجي (`repositories.py`, `migrations`).
- **interfaces (الواجهات):** يحتوي على طبقة الـ API والاتصال الخارجي (`serializers.py`, `views.py`, `urls.py`).

---

## 2. قواعد التعامل مع قاعدة البيانات وعزل المستأجرين (Multi-Tenancy)

- **الوراثة الموحدة:** يجب أن ترث جميع نماذج الجداول التشغيلية من `CombinedSharedModel` في `apps.shared.domain.models`.
- **العزل بـ `tenant_id`:** لا يُسمح بكتابة أي استعلام بدون استخدام الـ Custom Manager الخاص بعزل المستأجرين، أو استخلاص الـ Tenant من الـ request:
  ```python
  tenant_id = request.tenant.id if hasattr(request, 'tenant') else None
  ```
- **استخدام الـ UUID:** يُمنع استخدام المعرفات الرقمية المتسلسلة (Auto-increment IDs) للكيانات ويجب استخدام الـ UUIDv4 كمعرف أساسي لحماية خصوصية البيانات وصحة النقل.

---

## 3. معايير الأمن والتحقق (Security & Validation)

- **التحقق من المدخلات:** يتم استخدام الـ Serializers حصرياً للتحقق من البيانات مع تفعيل دوال التحقق المحددة لبلد العميل (مثل التحقق من الهوية الوطنية ورقم الجوال السعودي).
- **التدقيق الأمني (Audit Logs):** يجب إرسال حدث (Domain Event) عبر الـ Event Bus عند إجراء أي عملية حساسة لحفظها تلقائياً في سجل التدقيق الموحد.
