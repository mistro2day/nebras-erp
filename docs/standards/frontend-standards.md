# معايير تطوير الفرونت إند (frontend-standards.md) - Nebras ERP

تحدد هذه الوثيقة المعايير والقواعد الصارمة لبناء واجهات المستخدم وتكامل الفرونت إند باستخدام إطار عمل Angular 20.

---

## 1. الهيكل التنظيمي للموديول (Module Feature Structure)

يجب تقسيم أي موديول فرونت إند جديد وفق الهيكل التالي:

```
feature-name/
├── feature-name.routes.ts      # مسارات التوجيه والتنقل
├── dashboard/                 # لوحة التحكم الخاصة بالموديول
├── list/                      # شاشة عرض وجداول البيانات
├── details/                   # شاشة عرض التفاصيل والـ Timeline
└── create/                    # شاشات التسجيل والنماذج الديناميكية
```

---

## 2. إدارة الحالة واستخدام الـ Signals

- **الاعتماد على الـ Signals:** يُمنع استخدام المتغيرات الثابتة أو مكتبات إدارة الحالة المعقدة. يجب استخدام الـ Angular Signals لإدارة حالة المكونات والشاشات لضمان التحديث التلقائي الفوري والتوافق مع استهلاك الذاكرة.
- **التواصل مع السيرفر:** يجب عزل طلبات الـ HTTP بالكامل داخل Service مخصصة ترث من `ApiClientService` وتمرر البيانات للواجهة عبر Signals.

---

## 3. معايير الهوية البصرية والتوافق مع RTL

- **محاذاة اليمين إلى اليسار (RTL):** يجب تغليف وتنسيق جميع المكونات باتجاه RTL والاعتماد على خطوط وعناصر متوافقة مع اللغة العربية (مثل خط Cairo أو Outfit).
- **التوافق التام مع الشاشات (Responsive):** يجب تصميم كافة المكونات باستخدام Flexbox و Grid لتكون متوافقة مع الأجهزة اللوحية والهواتف المحمولة بنسبة 100%.
- **استخدام الـ Theme الموحد:** يتم قراءة الألوان والأكواد الخاصة بالمستأجر ديناميكياً من الـ `TenantService` وتطبيقها على المكونات.
