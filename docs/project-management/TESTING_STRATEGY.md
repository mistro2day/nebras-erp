# إستراتيجية الاختبارات والضمان الفني (TESTING_STRATEGY.md) - Nebras ERP

تحدد هذه الوثيقة إستراتيجية الاختبارات وأنواعها لضمان كفاءة وثبات المنصة مع توسعها.

---

## 1. مستويات الاختبارات (Testing Levels)

- **اختبارات الوحدة (Unit Tests):**
  - التركيز على منطق العمل وقوانين النطاق (`domain` & `application`) دون الارتباط بالشبكة أو قواعد البيانات.
- **اختبارات التكامل والـ API (Integration & API Tests):**
  - مراجعة سلامة الروابط وصياغة الردود الموحدة للنجاح والفشل والتحقق من الصلاحيات بـ `django.test.TestCase`.
- **اختبارات الواجهات (Frontend & Component Tests):**
  - التحقق من سلامة المكونات التفاعلية للفرونت إند Angular.

---

## 2. مستهدفات التغطية (Code Coverage Targets)

- تلتزم المنصة بمستهدف تغطية الكود البرمجي (Coverage) بنسبة لا تقل عن **80%** للمشاريع والـ APIs الحيوية (المصادقة، المالية، وعزل البيانات).
- تفعيل أدوات التحليل الآلي لتغطية الكود في خط الإنتاج وتجنب دمج أي ميزات لا تملك تغطية برمجية كافية.
