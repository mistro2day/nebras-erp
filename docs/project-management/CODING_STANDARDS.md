# معايير كتابة الشيفرات البرمجية (CODING_STANDARDS.md) - Nebras ERP

يوثق هذا المستند القواعد والاصطلاحات المعتمدة لكتابة وتنسيق الأكواد البرمجية لضمان أعلى مستويات النظافة البرمجية (Clean Code).

---

## 1. معايير لغة Python & Django

- **التنسيق:** الالتزام التام بـ **PEP 8**. يُنصح باستخدام أداة `black` أو `ruff` للتنسيق التلقائي.
- **التوثيق:** يجب كتابة Docstrings لجميع الوظائف والـ Classes والـ Methods لتوضيح هدفها والمدخلات والمخرجات.
- **عزل طبقة العمل:** لا تكتب منطق العمل (Business Logic) داخل الـ Views؛ يجب عزله داخل Use Cases أو Services في طبقة التطبيق (`application/`).

---

## 2. معايير لغة TypeScript & Angular

- **البرمجة التفاعلية:** الاستخدام الأمثل لـ **Angular Signals** لإدارة الحالات المحلية بكفاءة وعزل الـ Change Detection.
- **Standalone Components:** الالتزام بتصميم المكونات كـ Standalone Components لتقليل حجم الحزم البرمجية وتسهيل الـ Lazy Loading.
- **إدارة التنسيقات:** استخدام SCSS وتطبيق المتغيرات البصرية الموحدة في نظام التصميم (`design-system.scss`).

---

## 3. معايير واجهات الـ REST API

- **الردود القياسية:** الالتزام بصيغة الرد الموحدة (Standard Response Format) للنجاح والفشل.
- **تسمية المسارات:** استخدام صيغة الجمع وبأحرف صغيرة للكيانات (kebab-case) مثل: `api/v1/academics/academic-years/`.
