# موديول القبول والتسجيل (Admissions Management)

يوفر هذا الموديول واجهات وخدمات التقديم والالتحاق ومتابعة الطلبات للطلاب الجدد، مع دعم الاستمارة الإلكترونية الشاملة المطابقة للاستمارة الورقية الرسمية، وتعدد المدارس (بنين / بنات).

## 📋 استمارة التسجيل والقبول الشاملة (6 خطوات)
1. **البيانات الشخصية للتلميذ والأشقاء**: الاسم الرباعي، الميلاد (`nb-datepicker`)، الجنس، الرقم الوطني/الجواز، والأشقاء الميدانيين بالمدارس النموذجية وتحديد عددهم وقسمهم.
2. **بيانات ولي الأمر والبديل للطوارئ**: صلة القرابة، الاسم الرباعي، الرقم الوطني (المعتمد لربط بوابة ولي الأمر)، الهواتف، هاتف الأم، **ورقم واتساب المتابعة الدولي المعتمد E.164 مع مُنتخب مفتاح الدولة** (لإرسال الرسائل التلقائية عبر موديول الاتصالات)، وبيانات البديل الطارئ في حالة الغياب.
3. **البيانات الصحية والاجتماعية والترحيل**: المشاكل الصحية (تُربط بملف العيادة)، المشاكل الاجتماعية، وسيلة الترحيل (تُربط بالنقل)، وسُبل الاعتماد في المذاكرة.
4. **الدراسة السابقة والمستندات**: المدرسة السابقة والنسبة، وجاهزية رفع المستندات الـ 4 المحددة (صورتين، الشهادة السابقة، الرقم الوطني، وإثبات ولي الأمر).
5. **اللوائح والتعهدات المدرسية**: التزام باللوائح الإدارية (24 بنداً)، الأكاديمية (13 بنداً)، التنظيمية (10 بنود)، وتعهد حظر الهاتف النقال والأجهزة.
6. **الرسوم والمراجعة والمعاينة الرسمية الورقية**: تفصيل رسوم التسجيل (200,000 جنيه) والرسوم الدراسية (500,000 جنيه)، ومعاينة واستخراج الاستمارة الرسمية الورقية المطبوعة بضغطة زر.

## 🏫 هيكلة المدارس المتعددة وتخصيص الفروع (Boys & Girls Schools)
- **الفرز والتوجيه الذكي**: توجيه تلقائي عند اختيار الجنس (ذكر ← مدرسة البنين، أنثى ← مدرسة البنات) مع إمكانية التحديد الصريح بالفرع.
- **إعدادات القبول المخصصة لكل فرع**: إمكانية فتح/إغلاق التقديم لمدرسة البنين أو مدرسة البنات بشكل منفصل، وتحديد المقاعد الشاغرة بكل صف لمدرسة البنين مقابل مدرسة البنات.

## 📋 شاشة تفاصيل الطلب وإدارته الشاملة (Application Details View)
- **إظهار كافة بيانات الاستمارة الـ 6**:
  - بطاقة البيانات الشخصية والأكاديمية للتلميذ والفرع المستهدف (`👦 مدرسة البنين` / `👧 مدرسة البنات`).
  - بطاقة ولي الأمر الرئيسية المعززة بالرقم الوطني المعتمد لربطه ببوابة أولياء الأمور.
  - بطاقة الشخص البديل في حالة الغياب والطوارئ (الاسم، صلة القرابة، الهاتف، السكن).
  - بطاقة العيادة والملاحظات الصحية والتأثير الإرشادي الاجتماعي.
  - بطاقة قطاع النقل والترحيل الرسمي وسُبل الاعتماد في المذاكرة.
  - بطاقة إحصاء وتفاصيل الأشقاء المسجلين بالمورد النموذجية.
  - بطاقة إقرارات اللوائح المدرسية 4 (الإدارية، الأكاديمية، التنظيمية، وحظر الهواتف).
- **الربط المباشر لرسائل الواتساب الفورية عبر سيرفر الإيفولوشن (Evolution Baileys WA Engine Integration)**:
  - عند تغيير حالة الطلب (`قبول` / `رفض` / `قائمة انتظار` / `جدولة مقابلة`) أو النقر على زر `💬 إشعار الواتساب`:
  - اختيار نماذج المراسلة من موديول الاتصالات (`ADM_ACCEPTED`, `ADM_WAITLIST`, `ADM_REJECTED`, `ADM_INTERVIEW`).
  - الاستبدال التلقائي للمتغيرات الديناميكية (`{{guardian_name}}`, `{{student_name}}`, `{{application_number}}`, `{{school_name}}`).
  - إرسال فورى عبر خادم الإيفولوشن بايلس `CommunicationsService.sendMessage` وتوثيق الرسائل بسجلات موديول الاتصالات مع خيار التراسل الاحتياطي عبر رابط `https://wa.me/{whatsapp_phone}`.
- **معاينة وطباعة الاستمارة الورقية الرسمية المطابقة للأصل**:
  - زر `📄 المعاينة والطباعة الورقية` لتوليد واستخراج نسخة الـ PDF الرسمية للمؤسسة بضغطة زر.

## 🔄 محرك مسارات العمل الموحد (Workflow Engine)
تم تصميم المحرك العام في `apps/workflow` ليقوم بإدارة الحركات بين الحالات:
- مسودة (Draft)
- تم التقديم (Submitted)
- قيد المراجعة (Under Review)
- المقابلة مجدولة (Interview Scheduled)
- مقبول (Accepted) / مرفوض (Rejected)
- مُسجّل كطالب (Enrolled)
