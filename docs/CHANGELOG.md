# سجل التغييرات (Changelog) - Nebras ERP

جميع التغييرات والتحسينات المضافة للمنصة يتم توثيقها هنا تباعاً بالاعتماد على معايير الحوكمة ونسخ الإصدارات القياسية.

## [v1.6.0] - 2026-07-21

### Added
- **إعادة هيكلة استمارة القبول والتسجيل الشاملة (Admissions Management Rebuild):**
  - إعادة بناء شاشة التقديم العامة (`public-apply.component.ts`) والداخلي (`applicant-form.component.ts`) إلى معالج 6 خطوات متكامل ومطابق للاستمارة الرسمية الورقية ("المورد الجديدة للتعليم الخاص").
  - دعم حقول الأشقاء المسجلين بالمدارس، الرقم الوطني المعتمد لولي الأمر لربطه ببوابة ولي الأمر، التجهيز لربط ملف العيادة والترحيل، ودعم رفع المستندات الـ 4 المحددة.
  - إتاحة مكون الطباعة والمعاينة الرسمية المطابقة للنسخة الورقية `ApplicantPrintModalComponent`.
- **دعم تعدد المدارس (مدارس البنين والبنات) والفلترة التفاعلية بالداشبورد:**
  - تحديث نموذج الفرع `Branch` بالباك إند بدعم `school_gender_type` (`boys` / `girls` / `coed`) وتوليد الهجرة `0002_branch_school_gender_type.py`.
  - إضافة مبدّل الفروع المدرسية بالهيدر العلوي (`DashboardLayoutComponent`) لتمكين المدير العام والمستخدم الخارق من اختيار "جميع الفروع" أو "مدرسة البنين" أو "مدرسة البنات".
  - ربط الداشبورد الرئيسي (`dashboard.component.ts`) بإشارة `TenantService.activeBranch` للفلترة اللحظية والتفاعلية التامة للمؤشرات، ومسار القبول (Admissions Funnel)، والبيانات المالية.
- **تخصيص إعدادات القبول والتسجيل لكل فرع مدرسة:**
  - تطوير شاشة الإعدادات (`admission-settings.component.ts`) لدعم ضبط فتح/إغلاق التقديم والمقاعد الشاغرة للصفوف لمدرسة البنين ومدرسة البنات بشكل منفصل.
- **التوثيق:**
  - توثيق كافة التغييرات والهيكلة الجديدة في ملفات التوثيق الفني `docs/admissions.md` و `docs/modules/dashboard.md` و `docs/CHANGELOG.md`.

## [v1.4.0] - 2026-07-05

### Added
- **منصة الأتمتة المؤسسية (Enterprise Automation Platform):**
  - تأسيس تطبيق `automation_platform` للباك إند (نماذج مصمم مسارات العمل، مصمم القواعد، محرك الأتمتة، المنصة منخفضة الشيفرة، الإضافات، العمليات، DevOps) وتطبيق هجرة أولية موحّدة.
  - **مصمم مسارات العمل المرئي**: تحقق ومحاكاة ونشر يترجم المخطط إلى محرك مسارات العمل الحالي (`apps.workflow`) دون تكرار التنفيذ.
  - **مصمم القواعد**: جداول/أشجار قرار ومجموعات قواعد فوق محرك القواعد (`apps.rules`) مع محاكاة ونشر.
  - **محرك الأتمتة**: محفّزات (أحداث/جدولة/Webhook) وإجراءات تفوّض للمحركات القائمة وناقل الأحداث، مع سجل تشغيل وسياسة إعادة محاولة.
  - **منصة منخفضة الشيفرة**: مولّد شيفرة متوافق مع DDD (نماذج تمتد `CombinedSharedModel`، عروض تمتد `BaseCRUDViewSet`) مع مراجعة يدوية قبل التطبيق.
  - **العمليات وDevOps**: لقطات صحة النظام والتنبيهات، وواجهات تحضيرية لرايات الميزات والنشر والتراجع ووضع الصيانة (دون نشر سحابي).
  - **SDK** بايثون/تايبسكريبت + CLI، **تكامل مركز القيادة** (تسجيل أوامر آمن للمستأجر)، و**واجهة ذكاء اصطناعي قابلة للاستبدال** (بلا نموذج مضمّن).
  - مُقيّم تعبيرات آمن (AST sandbox) وواجهات REST تحت `/api/v1/automation/` وواجهة Angular تحت `/automation`.
  - إصلاح ربط `approval_center` المكسور (ملفات `__init__.py` و `urls.py` المفقودة) لاستعادة تحميل المشروع.
  - دمج وتفعيل اختبارات الكفاءة بنسبة نجاح 100% (39 اختباراً لمنصة الأتمتة).
